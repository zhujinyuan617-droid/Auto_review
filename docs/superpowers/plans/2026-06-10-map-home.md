# SP-Map 知识地图首页 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development 或主会话直接执行(本期模式:fable 主会话写核心,fable 子代理并行做互不相交的视图)。Steps 用 checkbox 跟踪。

**Goal:** 打开应用即见全屏知识地图(分区、镜头、核心度、着陆、特写),导航收敛 4 组,检索屏联动计数;全部机械计算零 AI。

**Architecture:** 桌面侧新增 `map/` 服务模块(读 elements_index.sqlite + 卡片 + authorship,算相似度/聚类/布局,缓存 JSON);3 个新端点 + 1 个检索增强端点;前端新 map 视图为首页,旧视图全保留。引擎只加 Task 0 一个独立脚本。

**Tech Stack:** Python(纯标准库算法:IDF 加权、标签传播聚类、FR 力导布局)、FastAPI、前端 vanilla JS + 既有 vendored force-graph(静态坐标模式)。

## Task 1(核心,主会话):map 服务 — 相似度/聚类/区名

**Files:** Create `desktop_app/src/autoreview_app/map/__init__.py`、`map/graph.py`;Test `tests/test_map_graph.py`

`graph.py` 纯函数(全部确定性,禁随机种子外的随机):
- `LENS_FACETS = {"topic": ("topic",), "method": ("preparation","measurement","simulation"), "material": ("material",)}`(time/institution 镜头不走要素相似度,见 Task 3/5)。
- `paper_features(db_path, lens) -> dict[paper_id, set[element_id]]`:SQL 取 role=used 的 canonical(经 resolve),按镜头 facet 过滤;topic 镜头改读卡片 classification.topic_ids。
- `idf(features) -> dict[element_id, float]`:log(N/df),df=含该要素论文数。
- `similarity_edges(features, idf, top_k=10, min_score=0.0)`:两篇共享要素 IDF 和;每篇保 top-k(与 candidate_edges 同配方)。
- `label_propagation(nodes, edges, iters=20) -> dict[paper_id, cluster_id]`:初始每点自标签;按 paper_id 排序同步更新,邻居标签按边权投票,平票取标签 id 小者;收敛或 iters 止——确定性。
- `name_clusters(members, features, idf, registry) -> {cluster_id: label}`:区内最高频要素,排除 idf 低于全局中位数 30 分位的"烂大街"项,取 top-1~2 的 display_name 连"·"。
- 测试:IDF 公式;top-k 截断;标签传播确定性(同输入两跑同输出)+ 两团假数据分成两区;区名排除低 idf。

## Task 2(核心,主会话):布局 + 缓存 + 增量落位

**Files:** Create `map/layout.py`;Test `tests/test_map_layout.py`

- `fr_layout(nodes, edges, clusters, iters=150, seed=42) -> {paper_id:(x,y)}`:Fruchterman-Reingold + 每点向本区质心的引力项(cluster gravity);`random.Random(seed)` 初始化;输出归一化到 [0,1]²。
- `incremental_place(cache_nodes, new_features...)`:新点初始位 = top-3 最近邻(相似度加权)质心 + 按 paper_id 哈希的确定性微偏;只对新点跑 30 轮局部弛豫(老点冻结)。
- 缓存:`data/elements/map_layout_<lens>.json` = {schema_version:"0.1.0", lens, fingerprint, nodes:[{id,x,y,cluster,size,lit}], clusters:[{id,label,n}]};fingerprint = sha256(参数+论文集+各篇要素集摘要),不匹配→全量重算;论文集是超集(只多新篇)→增量。size = 节点加权度(核心度)。lit:该镜头下此篇有无数据(无要素=灰点)。
- 测试:确定性(同输入同坐标);增量不动老点坐标;fingerprint 变更触发全算;lit 标记。

## Task 3(核心,主会话):API — /map、/map/arrivals、/map/relayout

**Files:** Create `map/service.py`;Modify `api.py`、`config.py`(map_cache_dir 属性);Test `tests/test_api_map.py`

- `GET /map?lens=topic|method|material|time|institution`:
  - topic/method/material:走 Task1+2(缓存优先);
  - time:nodes 直接给(无坐标聚类,前端按年代条带布点),payload 含 year;
  - institution:按 authorship 的 institution_ids 聚(同机构一区,>1 篇的机构才成区,孤篇归"其他");
  - 镜头枚举端点附带返回(从 LENS_FACETS + 内置 time/institution)。
- `GET /map/arrivals`:最近导入批次 = 按 literature_card.json 的 mtime 聚批(相邻间隔 ≤30 分钟为一批,取最新批);
  每篇附 top-3 最近邻(topic 镜头相似度)与所落区;无近邻(分数 0)标 `isolated:true`。
  ——批次定义是 mtime 启发式,**诚实标注在 docstring**;将来 importer 落 batch 标记后替换。
- `POST /map/relayout?lens=`:删缓存重算,返回新布局。
- 503 规约:要素索引未建 → method/material 镜头 503(沿用 _elements_db_or_503),topic/time 仍可用。
- 测试:5 镜头各回 200 + 形状;缓存命中(写探针:二次请求不重算——monkeypatch 计数);arrivals 批次聚类;relayout 删缓存。
